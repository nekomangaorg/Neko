## 2026-03-15 - [Optimize String Splitting in DB Cursor Mapping] **Learning:** [Database queries returning grouped concatenated strings (e.g. `GROUP_CONCAT` in SQLite) can cause massive CPU and GC overhead when mapped to UI models if the raw string is indiscriminately `split()` inside a tight loop.] **Action:** [Use zero-allocation fast paths with `indexOf` loops to calculate count when complex splitting/filtering isn't strictly necessary, and `String.contains` when simple substring existence checks are enough to prevent performance bottlenecks.]
