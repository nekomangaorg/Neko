## 2026-03-10 - [Added @Immutable to Library UI Data Classes] **Learning:** [Jetpack Compose marks data classes containing collections like `List` as unstable, which can cause severe recomposition issues in large lists like LibraryScreen. `PersistentList` alone might not be enough if the class itself isn't marked.] **Action:** [Always annotate data classes used in Lazy columns/grids with `@Immutable` if they contain collections or interface types to prevent performance bottlenecks.]
## 2026-03-10 - [Optimize Manga Backup History Queries] **Learning:** [When backing up manga history, the app was executing a DB query per history entry to retrieve the chapter URL. This created an N+1 query bottleneck within the database transaction.] **Action:** [Pre-fetch all chapters for a given manga with a single DB query, map them by ID, and use that map to assign URLs to history entries, thereby resolving the bottleneck.]
## 2026-03-10 - [Avoid Synchronous SharedPreferences Reads inside DefaultGetResolver] **Learning:** [Calling SharedPreferences repeatedly inside `mapFromCursor` of a database `GetResolver` causes massive N+1 performance bottlenecks during large queries (e.g., >1000 items). Caching the preference in the resolver avoids this, but requires avoiding singletons to prevent state corruption.] **Action:** [Always instantiate `DefaultGetResolver` classes dynamically per-query if they cache external state, and pre-fetch or lazily fetch necessary SharedPreferences once per query instance.]
## 2026-03-15 - [Avoid Intermediate Lists with mapNotNull] **Learning:** Using `.filter {}.map {}` chains on large lists allocates an intermediate list, which increases GC overhead and reduces performance on critical paths like resolving the next chapter. **Action:** Replace `.filter {}.map {}` chains with `.mapNotNull { if (...) ... else null }` to avoid intermediate list allocations and improve performance, especially when quickly jumping to reading.
