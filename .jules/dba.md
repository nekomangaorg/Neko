## 2026-04-30 - Fix Room Migration Silent Data Loss **Learning:** SQLite `CREATE TABLE IF NOT EXISTS ... AS SELECT` will silently skip the SELECT execution if the temporary table already exists from a previously interrupted migration, resulting in empty temporary tables during backup and eventual data loss when old tables are dropped. **Action:** Always explicitly drop temporary tables before attempting a backup during Room migrations and remove the `IF NOT EXISTS` clause so failures are loud rather than silent.
