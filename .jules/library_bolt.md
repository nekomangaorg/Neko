## 2026-03-10 - [Added @Immutable to Library UI Data Classes] **Learning:** [Jetpack Compose marks data classes containing collections like `List` as unstable, which can cause severe recomposition issues in large lists like LibraryScreen. `PersistentList` alone might not be enough if the class itself isn't marked.] **Action:** [Always annotate data classes used in Lazy columns/grids with `@Immutable` if they contain collections or interface types to prevent performance bottlenecks.]
## 2026-03-10 - [Optimize Manga Backup History Queries] **Learning:** [When backing up manga history, the app was executing a DB query per history entry to retrieve the chapter URL. This created an N+1 query bottleneck within the database transaction.] **Action:** [Pre-fetch all chapters for a given manga with a single DB query, map them by ID, and use that map to assign URLs to history entries, thereby resolving the bottleneck.]
## 2026-03-10 - [Avoid toMap() for Map Conversions] **Learning:** Using `.mapNotNull { ... to ... }.toMap()` instead of `.filter {}.associateBy {}` actually degraded performance by creating intermediate `Pair` objects. **Action:** Next time, avoid `.toMap()` chaining if the existing code efficiently uses `.associateBy {}`.
