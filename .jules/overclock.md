## 2026-03-10 - [Overclock: Cold Flow Sharing] **Learning:** Complex Kotlin Flow pipelines using `combine` and `map` repeatedly subscribe to intermediate flows. If those flows observe a database (`asFlow()`) or do expensive processing, multiple concurrent `combine` streams will execute the query redundantly per downstream subscriber. **Action:** Use `.shareIn(scope, SharingStarted.WhileSubscribed(), 1)` on shared, expensive intermediate flows (like `categoryListFlow`, `trackMapFlow`, and `filteredMangaListFlow`) to ensure they only spin up their heavy operation once, sharing the cached latest value to all parallel collectors.
